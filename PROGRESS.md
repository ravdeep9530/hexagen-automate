### Known issues from Day 4 (non-blocking)

1. **Stage 6 (Kimi-K2.6) ran out of `max_tokens=2048`** during reasoning — Dify's `azure_ai_studio` provider plugin caps `max_tokens` at 2048. The model spent all 2048 thinking and emitted empty `content`. Workarounds: (a) edit the plugin's parameter-rules YAML to raise the cap, (b) tighten the stage 6 system prompt to discourage long reasoning, (c) swap stage 6 to gpt-4.1 (less specialized for code but works).
2. **`resume_webhook_url`** stores `http://localhost:5678/...` because that's n8n's container-internal `WEBHOOK_URL`. The Express backend (Day 5) will rewrite `localhost` → `n8n` for internal calls, or `docker-vm-dev-01` for browser-facing calls.
3. **Duplicate workflows** from initial import attempts are inactive in n8n's list but couldn't be deleted (HTTP 400 from delete endpoint, likely past executions reference them). Cosmetic only.
4. **Wait node `httpMethod: POST`** is required (default is GET) for the approve POST to match. Captured in build_workflows.py.

## Decision log (deviations from original plan)

| Date | Decision | Reason |
|---|---|---|
| 2026-05-15 | Local Docker → Azure VM `docker-vm-dev-01` via Tailscale | Mac out of disk + RAM during Dify image pull |
| 2026-05-15 | 7 separate models → 3-model lineup (gpt-4.1 + DeepSeek-R1 + Kimi-K2.6) | User had only these 3 Foundry deployments |
| 2026-05-15 | Dify apps in `chat` mode, not `advanced-chat`/`workflow` | Simpler MVP; can convert specific stages later for tool use |
| 2026-05-15 | All 7 apps configured via Dify console API (not UI) | Faster, idempotent, replayable |
| 2026-05-15 | Foundry-served models capped at `max_tokens=2048` | Dify plugin default; raising requires plugin parameter-rules edit |
| 2026-05-15 | KB ingestion deferred from Day 2 | No real pilot repo picked yet |
| 2026-05-15 | n8n workflows defined as Python-generated JSON, imported via /rest API (cookie auth) | Reproducible + idempotent + lives in git |
| 2026-05-15 | server-generated `run_id` (UUID) instead of optional client override | Postgres node param-binding with optional empty value was unreliable |
| 2026-05-15 | Wait node `httpMethod: POST` explicitly (default is GET) | UI / approve flow always uses POST |
