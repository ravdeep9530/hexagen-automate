## Day 4 — n8n workflows 🔧

**Stage-template sub-workflow** (one node sequence, parameterized by `stage_name`):

1. **Execute Workflow Trigger** (entry, accepts `run_id`, `stage_name`, `inputs`)
2. **Postgres** — `SELECT api_key, app_id FROM dify_app_keys WHERE stage=$1`
3. **Postgres** — `UPDATE pipeline_stage_status SET status='running', started_at=now() WHERE run_id=$1 AND stage=$2`
4. **HTTP Request** — `POST http://dify-nginx/v1/chat-messages` with the app's API key, blocking mode
5. **Function** — strip `<think>...</think>` blocks (DeepSeek-R1 quirk), parse JSON, validate
6. **HTTP Request** — `PUT` artifact to SharePoint via MS Graph
7. **Postgres** — `UPDATE pipeline_stage_status SET status='awaiting_approval', dify_run_id=$1, artifact_url=$2, artifact_json=$3, resume_webhook_url=$4`
8. **Slack/Teams** node — post Approve/Reject card with `$resumeUrl` (optional for PoC; UI handles approvals)
9. **Wait** — `Resume: On Webhook Call`, 24h timeout
10. **IF** — approved? → return to parent; else → mark `rejected`, stop

**Parent `pipeline` workflow**: a 7-link chain calling the sub-workflow with `stage_name` = `requirements`, `optimize`, …, `test`. Stage 6 has a **Split In Batches** fan-out (one Implementation pass per Stage-5 ticket).

**Webhook trigger**: `POST /webhook/pipeline/start` accepting `{repo, raw_request, requester_id, run_id}`.
